Activity 2: What IS Data Analytics?
15 minutes

Simple Definition (on projector):

Data Analytics = Using numbers and patterns to make better decisions

Break It Down:

  1. Collect: Gather the numbers (stats from games)
  2. Analyze: Look for patterns (what works? what doesn't?)
  3. Decide: Make smarter choices (change strategy, improve training)

Real Sports Example Walkthrough (8 minutes):

Scenario: A basketball player wants to improve their scoring

Step 1 - Collect:

  • Track shots taken vs. shots made for 10 games
  • Record shot location (3-point, mid-range, layup)

Step 2 - Analyze:

  • Notice: 3-point shooting = 25% accuracy
  • Mid-range = 45% accuracy
  • Layups = 70% accuracy

Step 3 - Decide:

  • Focus on getting to the basket more
  • Practice 3-point shots more in training
  • Maybe avoid mid-range shots for now
Turn and Talk (3 minutes): "Think of a decision you make in your sport. What data could help you make that decision better?"
Have 2-3 students share their ideas with the whole group. Validate every response and show how it connects to data analytics.

Introduction to Google Colab (5 minutes):

Google Colab = A free, online notebook where you write and run code. No software to install. Works in your web browser just like Google Docs.

Step-by-Step Setup (Follow along with students):

Step 1: Open a web browser

Step 2: Go to colab.research.google.com

Step 3: Sign in with Google account (school account works)

Step 4: Click "New Notebook"

Walk around and help students who get stuck. Common issues: not signed into Google, blocked by school firewall (have IT help ready), confused by the interface.

Tour of the Interface (5 minutes):

Point out these key features on the projector:

  • Code Cell: Where you type your code (looks like a gray box)
  • Play Button: Click this to run your code
  • Output Area: Where results appear below the code
  • Notebook Name: Top left - rename it to "My Sports Analytics"
Tell students: "Think of this like a digital notebook. You write instructions (code), press play, and the computer follows your instructions."

Now click the play button (‚ñ∂Ô∏è)

Activity 2: Variables - Containers for Your Data
25 minutes

Concept Introduction (8 minutes):

Variable = A named container that stores information. Like a labeled box that holds your data.

Real-World Analogy:

Draw on whiteboard:

  • Box labeled "points" ‚Üí contains the number 24
  • Box labeled "player_name" ‚Üí contains "LeBron James"
  • Box labeled "position" ‚Üí contains "Forward"

"In coding, we create these 'boxes' and give them names. Then we can use those names to work with the data inside."

First Variable Together (7 minutes):

Type together on Google Colab:

# Create a variable points = 15 # Display the variable print(points)

Explain:

  • points = name of our box (container)
  • = means "put the following value into this box"
  • 15 = the value we're storing
Important: The equals sign (=) in coding means "assign" or "store", NOT "equals to" like in math class. This confuses students at first.

More Variable Practice (10 minutes):

Students type and run each example:

# Different types of variables # Numbers (integers) rebounds = 8 assists = 5 # Text (strings - always use quotes) player_name = "Maya" team = "Eagles" # Decimal numbers (floats) shooting_percentage = 0.456 # Display them all print(player_name) print("Rebounds:", rebounds) print("Assists:", assists) print("Shooting %:", shooting_percentage)
Check for Understanding: "Why do we use quotes around 'Maya' but not around 8?"
Answer: Text needs quotes. Numbers don't.
Encourage students to change the names and numbers to represent themselves or their favorite player!
Activity 3: Math with Code - Real Sports Stats
30 minutes

"Now we're going to use code to do what coaches do: calculate performance statistics!"

Basic Math Operations:

Calculating Points Per Game (10 minutes):

Work through this example together:

# Season statistics for a player total_points = 187 games_played = 10 # Calculate average points per game points_per_game = total_points / games_played # Display the result print("Points per game:", points_per_game)
Key Insight: "You just did what NBA analysts do! They calculate PPG (points per game) exactly like this."

More Real Stats Calculations (15 minutes):

Students type and run:

# Basketball shooting percentage shots_made = 45 shots_taken = 100 shooting_percentage = (shots_made / shots_taken) * 100 print("Shooting percentage:", shooting_percentage, "%") # Soccer goals per game total_goals = 12 games = 8 goals_per_game = total_goals / games print("Goals per game:", goals_per_game) # Track: Average race time race1 = 12.5 # seconds race2 = 12.3 race3 = 12.7 average_time = (race1 + race2 + race3) / 3 print("Average 100m time:", average_time, "seconds")
Walk around and help students debug. Common errors: forgetting parentheses, typos in variable names, forgetting quotes around text.
Quick Discussion: "Which of these stats would be most useful for YOUR sport? Why?"
